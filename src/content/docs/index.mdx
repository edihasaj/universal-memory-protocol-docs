## The third interoperability layer

Agents can already call tools (**MCP**) and talk to each other (**A2A**). What they
can't do is **carry memory across sessions, agents, and vendors**. Every harness
reinvents memory privately and non-portably. UMP fixes that.

<CardGrid>
  <Card title="Tools - MCP" icon="setting">
    Model Context Protocol standardized how agents call functions and read resources.
  </Card>
  <Card title="Coordination - A2A" icon="random">
    Agent2Agent standardized how agents discover and invoke one another.
  </Card>
  <Card title="Memory - UMP" icon="heart">
    Universal Memory Protocol standardizes how agents **remember** - portable knowledge across sessions, agents, and vendors.
  </Card>
</CardGrid>

## Why it's shaped like this

UMP does **not** invent a new wire protocol. The lesson of MCP is *minimal
primitives + ride existing rails + great SDKs + neutral governance*.

<CardGrid>
  <Card title="Rides MCP" icon="puzzle">
    The primary binding is an MCP profile (`ump.*` tools). Any MCP host - Claude
    Code, Codex - speaks UMP today, with zero new transport.
  </Card>
  <Card title="Six operations" icon="list-format">
    `capabilities · recall · remember · revise · forget · get` (+ optional
    `feedback`/`subscribe`). A conforming client is ~100 lines.
  </Card>
  <Card title="One portable record" icon="document">
    Typed, scoped, bi-temporal, signed JSON. Reuses W3C PROV + DID - no new vocabulary.
  </Card>
  <Card title="Adopt incrementally" icon="seti:config">
    Four conformance levels. Ship an `*.ump.json` export (L0) today; wire the full
    runtime (L3) when ready.
  </Card>
</CardGrid>

## What's standardized - and what isn't

UMP standardizes **structure, provenance, access, and trust**. It deliberately
leaves **intelligence** - extraction, ranking, decay, consolidation - to the
engine underneath, so implementations compete on quality while staying interoperable.

<CardGrid>
  <Card title="Bi-temporal, supersede-never-delete" icon="seti:clock">
    Valid-time + transaction-time. A fact that changes is closed and linked to its
    successor - not overwritten. The credible answer to staleness.
  </Card>
  <Card title="User-owned & signed" icon="approve-check">
    The operator - not the model vendor - holds the key. Records are content-addressed
    and signed with a DID. Portable, verifiable, no lock-in.
  </Card>
  <Card title="Injection-resistant by mandate" icon="warning">
    Memory is attacker-controllable input. The spec *requires* a verify → filter →
    frame rehydration pipeline. Never string-interpolated into the prompt.
  </Card>
  <Card title="Consent in the record" icon="setting">
    Retention, exportability, and redaction travel *with* the data and are enforced
    at every boundary.
  </Card>
</CardGrid>
